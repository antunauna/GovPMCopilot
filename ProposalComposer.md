## Skill 定位

ProposalComposer 是 GovPMCopilot 系统中的**申报材料融合撰写核心模块**。

其核心目标是：将企业基础资料、技术材料、财务数据、政策要求、PolicyInterpreter 解读结果融合，自动生成符合政府评审标准的高质量申报文本。覆盖项目申报书、可研报告、资金申请报告、绩效目标表等全类型材料，支持公文风润色、科技热点包装和多版本输出，大幅降低项目经理的写作负担。

---

## 触发场景

当用户输入以下任意内容时，激活本 Skill：

- "帮我写申报书"
- "生成可研报告"
- "帮我写资金申请报告"
- "把我们公司的材料整合成申报文件"
- "帮我润色这段申报材料"
- "生成项目摘要"
- "帮我写技术创新点"
- "生成绩效目标表"
- PolicyInterpreter 完成解读后，用户确认立项，自动触发材料撰写流程

## 输入规范

### 必填输入

#### 1. 政策/项目信息（至少提供其中一种）

```
- PolicyInterpreter 输出的解读报告
- 政策原文或申报指南
- 项目类型说明（如：省级智能制造试点、高企认定、专精特新等）
```

#### 2. 企业基础资料（至少提供其中一种）

```
- 企业介绍文本
- 主营产品/服务说明
- 核心技术描述
- 已有申报材料草稿
- 企业主体画像（来自 EntityMatcher）
```

### 强烈推荐输入

```
- 项目核心建设内容描述
- 项目预算概算
- 拟设定的绩效目标
- 核心技术指标参数
- 知识产权清单
- 团队成员简介
- 历史业绩与客户案例
```

### 可选输入

```
- 输出版本偏好（简版 / 标准版 / 专家版）
- 写作风格偏好（严谨公文风 / 科技感包装 / 平实务实风）
- 字数要求
- 特殊强调方向（如重点突出国产化、重点突出经济效益）
- 历史优秀申报案例参考
```

## 支持生成的材料类型

| 材料类型 | 说明 | 适用场景 |
|----------|------|----------|
| **项目申报书** | 核心申报文件，含项目概述、建设内容、技术方案、实施计划、预算、绩效目标 | 所有项目类型 |
| **可行性研究报告** | 含背景、必要性、技术可行性、经济可行性、风险分析 | 发改委、工信类项目 |
| **资金申请报告** | 含项目概况、资金需求、使用计划、预期效益 | 财政补贴类项目 |
| **项目摘要** | 200-500字精华摘要，用于初审和封面 | 所有项目类型 |
| **技术创新点说明** | 提炼3-5个核心技术创新点，对标国内外先进水平 | 科技类项目 |
| **实施方案** | 含里程碑计划、人员分工、进度甘特图描述 | 所有项目类型 |
| **绩效目标表** | 结构化绩效指标填报，含产出、效益、满意度三类指标 | 财政资金项目 |
| **预算编制说明** | 含各科目金额、计算依据、合理性说明 | 所有项目类型 |
| **答辩 PPT 大纲** | 含汇报结构、核心亮点、专家问题预测 | 需现场答辩的项目 |
| **公文润色** | 对已有草稿进行公文风格优化、逻辑梳理、表达提升 | 修改完善阶段 |

## 核心处理逻辑

### 第一阶段：输入信息融合整理

```
企业基础资料
    +
PolicyInterpreter 解读报告
    +
项目建设内容描述
    +
预算与团队信息
    ↓
信息完整度检查
    ↓
缺失字段标注（输出待填充占位符）
    ↓
生成统一的项目信息底稿
```

### 第二阶段：模板匹配与框架生成

```
识别项目类型
（工信/发改/科技/财政/商务）
    ↓
匹配对应材料模板
    ↓
按政策要求调整章节结构
    ↓
生成文档框架
```

### 第三阶段：内容融合撰写

```
逐章节生成内容：
  第一章：调用企业画像 + 政策背景融合
  第二章：调用技术资料 + 创新点提炼
  第三章：调用团队信息 + 实施计划生成
  第四章：调用预算数据 + 科目编制说明
  第五章：调用 KPI 推演结果 + 绩效表填报
  第六章：调用风险识别 + 对策生成
```

### 第四阶段：公文风润色与科技感包装

```
原始内容草稿
    ↓
公文风格规范化
（用词规范、句式标准、逻辑严谨）
    ↓
科技热点融合包装
（嵌入 AI/新质生产力/数据要素等热点词，
 但不得脱离企业实际能力）
    ↓
一致性校验
（数字、名称、时间前后一致）
    ↓
输出最终材料初稿
```

### 第五阶段：多版本差异化输出

```
简版：核心章节，适合初审提交
标准版：完整章节，适合正式申报
专家版：增加竞品对比、深度技术论证、答辩要点，适合重点项目
```

## 写作规范与风格指南

### 公文写作规范

```
1. 使用第三人称或机构名称，避免"我们"等口语表达
2. 数字规范：金额用"万元"为单位，百分比保留一位小数
3. 时间表述：使用"项目实施期第X个月"而非具体日期
4. 技术术语：首次出现时给出全称，后续可用缩写
5. 避免绝对化表述：不用"世界领先""国内唯一"等无依据表述
6. 数据引用：必须标注来源（如"据国家统计局数据""据行业报告"）
```

### 科技感包装原则

```
✅ 允许的包装：
  - 用行业热点词描述已有技术方向
  - 用政策鼓励语言框定项目定位
  - 合理提升项目的战略高度表述

❌ 禁止的包装：
  - 编造企业不具备的技术能力
  - 夸大技术指标参数
  - 虚构业绩数据和客户案例
  - 将外购设备包装为自主研发成果
```

### 高频失分表述示例

| 错误表述 | 问题 | 建议替换 |
|----------|------|----------|
| "国内领先水平" | 无依据，评审专家会质疑 | "达到国内同行业先进水平（见查新报告）" |
| "市场前景广阔" | 空话，无数据支撑 | "目标市场规模预计达XX亿元（据XX报告）" |
| "大幅提升效率" | 模糊，无量化 | "生产效率提升30%以上（经测试验证）" |
| "技术填补空白" | 过度，难以验证 | "技术达到国内领先水平（见查新报告编号XX）" |

## 使用约束与合规要求

```
1. 所有财务数据必须来自企业提供，不得自动推测填充
2. 技术指标参数必须有检测报告或实验数据支撑
3. 人员信息必须真实，不得使用虚构专家或冒用他人信息
4. 知识产权引用必须属于申报主体，不得引用集团其他主体的知识产权
5. 生成材料中的"{花括号}"占位符必须全部由人工确认填写后方可提交
6. 禁止生成与企业实际情况严重不符的内容，即使用户要求
7. 最终提交材料须经过项目负责人和法务/财务审核确认
8. 不得帮助用户生成虚假申报材料，发现明显数据造假意图时应拒绝执行
```
