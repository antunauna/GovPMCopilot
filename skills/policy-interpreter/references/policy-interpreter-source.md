## Skill 定位

PolicyInterpreter 是 GovPMCopilot 系统中的**政策深度解读与申报策略生成模块**。

其核心目标是：将晦涩的政策原文转化为企业可直接执行的申报策略，从宏观背景、技术导向、标杆场景、资金池预判、申报路径到合规红线，全维度输出结构化解读报告，帮助项目经理快速理解"这个政策是什么、能给我什么、我该怎么报"。

---

## 触发场景

当用户输入以下任意内容时，激活本 Skill：

- "帮我解读一下这个政策"
- "这个政策到底支持什么方向？"
- "我们公司能从这个政策拿到多少补贴？"
- "这个项目的验收指标一般怎么设？"
- "申报这个项目有什么坑要避开？"
- "帮我分析一下这个政策的战略意图"
- 输入政策原文/链接/摘要，要求生成解读报告
- EntityMatcher 完成主体推荐后，自动触发对应政策的深度解读

---

## 输入规范

### 必填输入

#### 1. 政策信息（至少提供其中一种）

```
- 政策原文（直接粘贴）
- 政策摘要（结构化字段）
- 政策链接（需配合 PolicyRadar 抓取全文）
- 已结构化的政策对象（JSON）
```

#### 2. 企业/行业背景（可选但强烈推荐）

```
- 企业主营业务方向
- 所属行业
- 核心技术标签
- 已推荐申报主体（来自 EntityMatcher 输出）
```

### 可选输入

- 解读深度偏好（快速摘要 / 标准解读 / 深度战略分析）
- 重点关注维度（资金 / 技术方向 / 验收指标 / 申报材料）
- 对标历史项目（用于生成差异化申报策略）

## 核心处理逻辑

### 第一阶段：政策全文解析与结构化

```
输入政策原文
    ↓
提取基础元数据
（标题/机构/日期/地区/层级/截止时间）
    ↓
识别文种类型
（通知/指南/实施细则/申报书/奖补办法）
    ↓
分段结构化
（背景段/目标段/条件段/支持方式段/材料段/时间段）
    ↓
输出结构化政策对象
```

### 第二阶段：六维解读生成

对结构化政策对象，逐维度生成解读内容：

```
维度1：宏观背景与战略意图
  → 关联上位政策、国家战略、五年规划
  → 识别部门意图与考核压力

维度2：核心技术导向解码
  → 提取技术关键词
  → 映射到产业热点
  → 识别隐性偏好

维度3：标杆应用场景映射
  → 从案例、附件、评分标准中推断
  → 结合行业背景生成差异化场景

维度4：地方配套与资金补贴池预判
  → 识别资金来源与拨付方式
  → 联动同期配套政策数据库
  → 生成综合资金池预估

维度5：黄金申报路径与验收KPI推演
  → 推演标准申报流程
  → 生成KPI建议值区间
  → 标注关键节点与风险

维度6：申报合规红线与避坑指南
  → 提取政策硬性禁止条款
  → 结合历史案例归纳失败原因
  → 生成本政策特有风险提示
```

### 第三阶段：科技热点融合包装建议

```
识别政策鼓励的技术热点词
    ↓
匹配企业现有技术能力
    ↓
生成"热点词 + 企业能力"融合包装建议
（例：将"工业数据采集系统"包装为"工业互联网+AI质检数据要素平台"）
    ↓
输出差异化技术定位建议
```

### 第四阶段：推荐申报项目池生成

```
综合政策支持方向 + 企业技术标签 + 历史申报数据
    ↓
生成候选项目方向列表
    ↓
按匹配度、资金体量、申报难度评分排序
    ↓
输出优先级排序的推荐项目池
```

## 解读深度模式

根据用户需求，支持三种解读深度：

| 模式 | 适用场景 | 输出内容 | 生成时长 |
|------|----------|----------|----------|
| **快速摘要** | 初步筛选，判断是否值得深入 | 政策要点 + 是否推荐申报 + 一句话结论 | 短 |
| **标准解读** | 项目立项决策 | 六维完整解读 + 推荐项目池 | 中 |
| **深度战略分析** | 重点项目全力冲刺 | 六维解读 + KPI推演 + 竞品分析 + 差异化策略 + 答辩要点预测 | 长 |

默认模式：**标准解读**

## 使用约束与合规要求

```
1. 所有政策原文引用必须标注来源，不得凭空生成政策条款
2. 资金金额必须来自政策原文，不得推测或捏造补贴金额
3. 验收 KPI 建议值必须标注"建议值/参考值"，不得作为承诺
4. 科技热点融合包装建议不得与企业实际技术能力严重背离
5. 合规红线内容必须完整输出，不得省略或弱化
6. 当政策原文信息不足以支撑某维度解读时，必须标注"信息不足/需进一步验证"
7. 输出报告不得替代专业申报顾问的判断，最终申报策略须由人工审核确认
```

## 与其他 Skill 的协作关系

```
PolicyRadar ──→ PolicyInterpreter ──→ ProposalComposer
（提供政策全文）   （输出解读报告）      （材料撰写）
      ↑                  ↑
EntityMatcher ──────────┘
（提供推荐主体，触发定向解读）
                         │
                         ↓
                   ReviewAuditor
                   （评审复核）
```

### 上游依赖

| Skill | 提供内容 | 是否必须 |
|-------|----------|----------|
| PolicyRadar | 政策全文、结构化元数据 | 推荐（可手动输入替代） |
| EntityMatcher | 推荐申报主体、企业画像 | 可选（有则生成定向解读） |

### 下游输出

| Skill | 使用内容 |
|-------|----------|
| ProposalComposer | 技术方向定位、KPI建议值、项目包装方向 |
| ReviewAuditor | 合规红线清单、政策符合性评估基准 |

## 示例对话

**用户：**

**PolicyInterpreter 输出：**

```

政策名称：广东省2026年智能制造试点示范申报通知

一、宏观背景与战略意图

二、核心技术导向解码

1. 机器视觉与质检自动化 ← 与贵司高度匹配 ⭐

六、合规红线
❌ 已在其他省级项目获得同类补贴的，不得重复申报
❌ 项目不得早于申报通知发布日期开工...

推荐申报方向：

```
