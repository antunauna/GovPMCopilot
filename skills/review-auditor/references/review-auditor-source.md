## Skill 定位

ReviewAuditor 是 GovPMCopilot 系统中的**申报材料预审与评审复核模块**。

其核心目标是：站在政府评审专家的视角，对 ProposalComposer 生成或人工撰写的申报材料进行全面预审，输出政策符合性、预算合理性、技术可行性、绩效合理性等多维评分与修改建议，帮助项目经理在正式提交前发现并修复关键缺陷，最大化申报成功率。

---

## 触发场景

当用户输入以下任意内容时，激活本 Skill：

- "帮我审一下这份申报材料"
- "模拟专家评审"
- "这份申报书有什么问题？"
- "帮我找找申报书的漏洞"
- "预算这样写合理吗？"
- "绩效目标设置合不合适？"
- "提交前帮我检查一遍"
- ProposalComposer 生成初稿后，用户确认进入预审流程，自动触发

---

## 输入规范

### 必填输入

#### 1. 申报材料（至少提供其中一种）

```
- 申报材料全文（直接粘贴）
- ProposalComposer 生成的材料初稿
- 分章节上传的材料片段
```

#### 2. 政策/评审标准（至少提供其中一种）

```
- PolicyInterpreter 输出的解读报告（含合规红线）
- 政策原文或申报指南
- 评分标准/评审细则（如有）
```

### 可选输入

```
- 企业主体画像（来自 EntityMatcher，用于一致性校验）
- 历史评审意见（用于针对性强化）
- 审核重点偏好（如重点审预算 / 重点审技术）
- 评审专家背景偏好（工信口 / 财政口 / 科技口）
```

## 核心处理逻辑

### 第一阶段：材料解析与结构识别

```
输入申报材料全文
    ↓
识别文档结构（章节划分）
    ↓
提取关键数据节点
（金额/指标/时间/人员/资质/技术参数）
    ↓
与政策要求对照建立审核基准
```

### 第二阶段：六维并行审查

```
维度1：政策符合性
  → 合规红线一票否决核查
  → 申报主体资格复核
  → 政策导向契合度评估

维度2：技术可行性
  → 技术方案完整性评估
  → 创新点支撑充分度核查
  → 团队资质匹配度评估

维度3：预算合理性
  → 科目结构合规性核查
  → 金额合理性横向对比
  → 计算依据完整性检查

维度4：绩效目标合理性
  → 指标完整性核查
  → 指标值合理性评估
  → 指标可核查性检验

维度5：材料完整性
  → 章节完整性核查
  → 附件清单核查
  → 数据一致性校验

维度6：风险可控性
  → 风险识别完整性
  → 应对措施可信度
```

### 第三阶段：评分与问题分级

```
各维度独立评分
    ↓
问题严重度分级
（P0必改 / P1建议改 / P2可选优化）
    ↓
生成综合得分
    ↓
输出修改优先级行动清单
```

### 第四阶段：答辩问题预测

```
识别材料中的薄弱点
    ↓
结合历史评审问题库
    ↓
模拟评审专家视角
    ↓
生成高风险追问问题 + 建议回答方向
```

## 评审专家视角模拟

ReviewAuditor 内置三类评审专家视角，可根据项目类型自动切换：

| 专家类型 | 适用项目 | 审查侧重点 |
|----------|----------|-----------|
| **工信口专家** | 智能制造、技术改造、工业互联网 | 技术创新性、产业带动性、制造业转型贡献 |
| **财政口专家** | 所有财政资金项目 | 预算合理性、资金使用规范性、绩效目标可达性 |
| **科技口专家** | 研发类、高企、科技专项 | 研发投入强度、知识产权产出、技术先进性 |

---

## 问题严重度分级标准

| 级别 | 标识 | 定义 | 处理要求 |
|------|------|------|----------|
| 一票否决 | 🚫 | 触碰合规红线，提交即被否决 | 立即修正，否则不得提交 |
| P0 必改 | 🔴 | 严重影响评审得分或引发质疑 | 提交前必须完成修改 |
| P1 建议改 | 🟡 | 影响材料质量，有改进空间 | 尽量修改，提升竞争力 |
| P2 可选优化 | 🟢 | 锦上添花，不影响基本质量 | 时间允许时优化 |

## 数据一致性核查矩阵

ReviewAuditor 自动执行以下跨章节数据一致性核查：

```
营收数据：第一章基本情况 ←→ 第四章预算自筹说明 ←→ 附件审计报告
研发投入：第一章基本情况 ←→ 第五章绩效目标 ←→ 附件财务数据
专利数量：第一章基本情况 ←→ 第二章创新点支撑 ←→ 附件专利清单
团队规模：第三章团队介绍 ←→ 第四章人员费用 ←→ 附件社保证明
项目预算：第四章预算表 ←→ 封面申报金额 ←→ 资金申请报告
建设内容：第二章建设内容 ←→ 第四章预算科目 ←→ 第五章绩效目标
```

## 使用约束与合规要求

```
1. ReviewAuditor 仅对材料内容进行预审，不代表官方评审结论
2. 一票否决项核查结果为必须人工复核确认的高风险判断
3. 预算合理性横向对比数据为参考值，以政策规定上限为准
4. 答辩问题预测为模拟推断，不代表实际评审专家的提问
5. 评分结果不得用于对外宣传或作为申报成功的承诺依据
6. 发现明显数据造假或虚假申报内容时，拒绝出具审核通过意见
```

## 与其他 Skill 的协作关系

```
PolicyRadar
    ↓
EntityMatcher → PolicyInterpreter → ProposalComposer → ReviewAuditor
                                     （材料初稿）        （预审+修改建议）
                                                              ↓
                                                    （修改后回到 ProposalComposer）
                                                              ↓
                                                        最终提交版本
```

### 上游依赖

| Skill | 提供内容 | 是否必须 |
|-------|----------|----------|
| ProposalComposer | 申报材料初稿 | 必须（或人工上传） |
| PolicyInterpreter | 合规红线清单、政策符合性评估基准 | 强烈推荐 |
| EntityMatcher | 主体画像，用于一致性校验 | 推荐 |

### 下游输出

```
ReviewAuditor 输出修改意见
    ↓
项目经理确认修改项
    ↓
返回 ProposalComposer 进行定向修改
    ↓
再次触发 ReviewAuditor 复审（循环直至通过）
    ↓
输出最终提交版本
```
